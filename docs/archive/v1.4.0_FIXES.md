### 3. Ctrl+F Filter Navigation ⚠️ BROWSER LIMITATION

**Issue:** When using Ctrl+F to focus filter dropdown, arrow keys change selection but then close dropdown and move focus away

**Root Cause:**
- This is **standard browser behavior** for native `<select>` elements
- When a `<select>` is focused, arrow keys change the selection AND close the dropdown
- After selection changes, focus moves to next focusable element
- This is how native HTML select elements work across all browsers

**Attempted Solutions:**
- ❌ Dispatching `mousedown` event - Opens dropdown but arrow keys still close it
- ❌ Preventing default on arrow keys - Breaks normal select functionality
- ❌ Capturing focus - Not possible with native select elements

**Current Behavior:**
- ✅ Ctrl+F successfully focuses the ISM filter dropdown
- ⚠️ Arrow keys change selection but close dropdown (native browser behavior)
- ℹ️ Toast message now explains: "Click to navigate with arrows"

**Workaround:**
- Press Ctrl+F to focus filter
- **Click the dropdown** to open it manually
- Now arrow keys work to navigate options without closing

**Future Solution:**
- Implement custom dropdown component (not native `<select>`)
- Full keyboard control with arrow navigation
- Added to ROADMAP under "Medium-term" improvements

**Files Modified:**
- `js/keyboard.js` - Updated toast message to explain browser limitation

### 4. Unresponsive Delete Button ✅ RESOLVED

**Issue:** User reported a specific row where the delete (X) button was unresponsive

**Root Cause:**
- Data corruption from the removed Ctrl+D duplicate function
- The duplicate function created entries with invalid or duplicate IDs
- Corrupted data prevented proper event handler attachment

**Resolution:**
- ✅ User cleared localStorage to remove corrupted data
- ✅ Ctrl+D function already removed to prevent future occurrences
- ✅ Fresh data state resolved the issue

**Lesson Learned:**
- The bug was in the data, not the current code
- Removing Ctrl+D prevents this type of corruption going forward
- Future: Consider adding data validation/repair utilities
